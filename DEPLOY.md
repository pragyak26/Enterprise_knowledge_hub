# Deploying for free (Render + Vercel)

This deploys the **backend to Render** and the **frontend to Vercel** — both free.

> ⚠️ Render's free tier has an **ephemeral disk** and **sleeps after ~15 min idle**
> (first request after sleep takes ~30–60s). Uploaded documents, the SQLite DB,
> and ChromaDB vectors **reset on every restart**. That's fine for a demo — just
> re-upload. For persistence, use a paid disk or a hosted Postgres + Chroma.

---

## 1. Backend → Render

1. Push your repo to GitHub (already done).
2. Go to https://dashboard.render.com → **New** → **Blueprint**.
3. Connect your GitHub repo. Render detects `render.yaml` and proposes the
   `ekh-backend` service.
4. Click **Apply**. Then open the service → **Environment** and set:
   - `GEMINI_API_KEY` → your Gemini key
   - `ALLOWED_ORIGINS` → leave empty for now (defaults to `*`); after step 2
     you can lock it to your Vercel URL.
   - `SECRET_KEY` is auto-generated by Render — leave it.
5. Wait for the deploy. Your API will be at:
   `https://ekh-backend-XXXX.onrender.com`
   Test it: open `…/health` → should return `{"status":"ok"}`, and `…/docs`
   for the Swagger UI.

> No `render.yaml`? You can also do **New → Web Service** manually:
> - Build command: `pip install -r requirements.txt`
> - Start command: `uvicorn app.main:app --host 0.0.0.0 --port $PORT`
> - Add the same env vars, plus `PYTHON_VERSION=3.13.4`.

---

## 2. Frontend → Vercel

1. Go to https://vercel.com → **Add New** → **Project** → import the same repo.
2. **Important:** set **Root Directory** to `frontend`.
3. Framework preset: **Vite** (auto-detected). Build command `npm run build`,
   output `dist` (already in `vercel.json`).
4. Under **Environment Variables**, add:
   - `VITE_API_BASE` → your Render backend URL (e.g.
     `https://ekh-backend-XXXX.onrender.com`)  — **no trailing slash**.
5. **Deploy.** Your UI will be at `https://your-app.vercel.app`.

---

## 3. Lock down CORS (recommended)

Back in Render → `ekh-backend` → **Environment**, set:

```
ALLOWED_ORIGINS = https://your-app.vercel.app
```

Save (Render redeploys). Now only your frontend can call the API.

---

## How it fits together

```
Browser → https://your-app.vercel.app   (React static, Vercel)
            │  fetch(VITE_API_BASE + "/auth/login" ...)
            ▼
        https://ekh-backend-XXXX.onrender.com   (FastAPI, Render)
            → Gemini API + ChromaDB (in-container)
```

Locally nothing changes: `npm run dev` still proxies `/api` to
`http://127.0.0.1:8000`, because `VITE_API_BASE` is only set in production.

---

## Troubleshooting

- **CORS error in browser console** → `ALLOWED_ORIGINS` on Render doesn't match
  your exact Vercel URL (check https/trailing slash).
- **Login/upload hangs ~30s then works** → Render free instance was asleep;
  it spun up. Normal.
- **502 / out of memory on Render** → ChromaDB is memory-heavy; the free 512MB
  can be tight. Upgrade the instance or trim dependencies.
- **Data disappeared** → expected on free Render (ephemeral disk). Re-upload, or
  move to Postgres (`DATABASE_URL`) + a hosted vector DB.
